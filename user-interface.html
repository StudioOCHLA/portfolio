A User Interface Concept

The development of two different design concepts for a website is part of this academic project, where the customer is not sure if they want much or a little color. Two graphic orientations/extremes are presented in the concept proposals. Developing two design concepts for a website is part of this university project. The main objective is to present my portfolio with some of my selected skills mainly in graphic design, web coding and the design of various user-friendly solutions.

Objective

My main objective is to present my portfolio with some of my selected skills mainly in graphic design, web coding and the design of various user-friendly solutions.

The objective is to create a website that is unified and has clear text, readable size, and color options that are suitable for individuals with different types of color blindness. I used TPGi's Colour Contrast Analyser (CCA) software, which allows the creation of accessible software.

Navigation menu

Four menu choices are used in the page's primary navigation. To make it clear what content the website contains, regardless of concept, and ensure implicitity.



  • Home tab, which leads to the home page
  • About tab that introduces the creator of the webpage
  • Project tab, which has a sub-menu with various categories
  • Contact tab, to offer the user the possibility of contact

hamburger menu

I chose to use a hamburger menu in the mobile versions of both concepts. In a fixed menu like the one in the laptop version, it would be difficult to see the menu choices. My decision on the size of the hamburger menu was based on Hurff's suggestion (WHAT?).

Project submenu

Under the Projects menu option, there are sub-pages for illustration, user experience, user interface, interaction design, and web development. Offering these choices already in the main menu would be a mistake because it could be perceived as messy and make the web site hard to navigate. Instead, they are placed in the same frame in a sub-menu at a regular interval to visually show their affinity.

Pop-up windows

The pop-up pages were sorted depending on the content. In some projects, certain content demands more space and has been divided into multiple pages.

To enhance accessibility and overview in the mobile prototypes, I have chosen to not present all the content, but rather to refer to the laptop version for additional information. The reason for this is that not all text and images work well in a small format. Additionally, the mobile version lacks most of the white space that was utilized on the laptop version.

On each project page, a description and title for the given project are arranged in a uniform grid. A subtle white space is used to separate these so that the user can easily identify where to continue reading. The descriptions are available in individual boxes, with a heading that makes it clear what the text is about.

Prototypes

The interface of the two concepts differs, which is a natural result of their different extremes. Concept 1's laptop prototype is accessible via a link, and the same applies to the mobile version. A link can also be used to access the second concept.

In terms of prototype designs, I worked in the Figma prototype program. I initially envisioned using Adobe XD for my second concept, but the program's limitations prevented me from doing so. I use animated GIF for the logo on the front page and Adobe XD does not support that.

Technical details

I searched for information online about the appropriate measurements to create a web page with commonly used proportions. According to GBKSoft's blog post on designing user-friendly digital products, 1366 x 768 pixels are the most commonly expected screen resolution on computer screens globally.

The most common width for mobile phones is 360 pixels internationally, but it is recommended to use as many as 375 pixels. I settled on a resolution of 375 pixels in width and 667 pixels in height for the mobile version. The width is closer to 414 pixels, which is the most commonly used width in both the US and England.

Uniformity, text and white space

All pages have a descriptive, centered and large title to clarify for the user where they are located on the website. I decided to use uniform text by using identical font and size for each concept's heading and all body text.

The banner

To advertise the Affinity brand, I created a banner on the website that can be clicked on. The design was created using Affinity Designer software and was based on their original logo. I combined text from IBM Plex Sans and Kohinoor Gujarati fonts from ITF to reproduce the original font. The word Designer under Affinity is similar to the expression in the original logo of the company where they used the font FS Koopman Medium.

The background image was legally obtained from Dan-Cristian Pădureț on Unsplash. I have decided to use a horizontal banner format for the laptop versions of the concepts, while using a portrait for the mobile version. The sender of the banner and their intended sale should be made clear to the receiver.

Image of the banner.

What I learned

It was interesting to explore how the same content may appear different depending on the structure and graphic design of a Web site. The appearance of a website plays a major role in how content is perceived, and it feels even more important to have a functioning structure.

Exploring more extremes in further work that is even more radical would have been interesting.

The content of the website has been adapted according to the format in which it is displayed. The mobile version has been simplified in both content and structure, but some functions, like downloading a PDF file, are not included.

Although the banner has been placed discreetly, it is still noticeable. I felt that this was successful because I aimed for it to not stand out too much and take up too much space in the field of view.
